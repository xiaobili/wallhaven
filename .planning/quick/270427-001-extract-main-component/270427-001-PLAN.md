# Quick Task: 提取 LocalWallpaper Main 组件

## 目标

将 `LocalWallpaper.vue` 中的 `<main>` 节点及其内容提取为独立组件 `LocalWallpaperMain.vue`，提高代码复用性和可维护性。

## 分析

当前 `LocalWallpaper.vue` 结构：
- 外层 `<div class="page-container">` 包含 Alert 和 ImagePreview
- `<main id="main">` 包含工具栏、壁纸列表、空状态、加载状态
- `<script>` 包含所有业务逻辑
- `<style>` 包含 main 相关样式

## 执行计划

### Task 1: 创建 LocalWallpaperMain.vue 组件

**Files:**
- `src/components/LocalWallpaperMain.vue` (新建)

**Action:**
创建新组件，包含：
- `<main>` 标签及其内部所有内容（工具栏、壁纸列表、空状态、加载状态）
- 通过 props 接收数据：`localWallpapers`, `loading`, `downloadPath`
- 通过 emits 暴露事件：`refresh`, `open-folder`, `set-wallpaper`, `delete-wallpaper`, `preview`
- 迁移相关的样式

**Verify:**
- 组件文件创建成功
- Props 和 emits 定义正确

---

### Task 2: 重构 LocalWallpaper.vue

**Files:**
- `src/views/LocalWallpaper.vue` (修改)

**Action:**
- 移除 `<main>` 节点，替换为 `<LocalWallpaperMain>` 组件
- 保留 `page-container` div、Alert 和 ImagePreview
- 保留业务逻辑在 view 层（组合式函数调用）
- 通过 props 传递数据，通过 events 处理子组件事件

**Verify:**
- 页面渲染正常
- 所有功能保持不变

---

### Task 3: 更新样式引用

**Files:**
- `src/components/LocalWallpaperMain.vue` (修改)

**Action:**
- 将 `.local-wallpaper-page` 及相关样式迁移到新组件
- 确保 `list.css` 引用正确

**Verify:**
- 样式无变化
- 响应式设计正常

---

## 预期结果

- `LocalWallpaperMain.vue` 组件可复用
- `LocalWallpaper.vue` 更简洁，专注于页面级逻辑
- 功能和样式完全保持不变
