# Quick Task Plan: CollectionDropdown Animation

**Created**: 2026-04-28
**Task**: Add macOS-style minimize animation to CollectionDropdown open/close transitions

---

## Context

The current `CollectionDropdown.vue` uses `v-if` for visibility control, which results in instant show/hide without any transition. The goal is to add a macOS-like minimize animation (scale + fade + slight movement) for a polished user experience.

---

## Tasks

### Task 1: Implement Vue Transition with macOS-style Animation

**File**: `src/components/favorites/CollectionDropdown.vue`

**Changes**:
1. Replace `v-if` with Vue's `<Transition>` component
2. Add CSS keyframe animations for:
   - **Open (enter)**: Scale from 0.8 to 1, fade in, slight downward slide
   - **Close (leave)**: Scale from 1 to 0.8, fade out, slight upward slide
3. Use `transform-origin` based on dropdown position (top-left corner near cursor)
4. Animation duration: ~200ms with ease-out timing

**Animation CSS Classes**:
```css
.dropdown-enter-active {
  animation: dropdown-open 0.2s ease-out;
}

.dropdown-leave-active {
  animation: dropdown-close 0.15s ease-in;
}

@keyframes dropdown-open {
  from {
    opacity: 0;
    transform: scale(0.8) translateY(-8px);
  }
  to {
    opacity: 1;
    transform: scale(1) translateY(0);
  }
}

@keyframes dropdown-close {
  from {
    opacity: 1;
    transform: scale(1) translateY(0);
  }
  to {
    opacity: 0;
    transform: scale(0.8) translateY(-8px);
  }
}
```

---

## Implementation Notes

- Transform origin should be `top left` so the animation appears to emerge from where the user clicked
- Keep the animation fast (< 250ms total) to feel responsive
- The existing `dropdownStyle` computed property already sets position, so no changes needed there
- The Teleport wrapper should remain outside the Transition

---

## Acceptance Criteria

- [ ] Dropdown animates in with scale + fade + slide on open
- [ ] Dropdown animates out with reverse animation on close
- [ ] Animation feels smooth and macOS-like
- [ ] No visual glitches during transition
- [ ] Performance is smooth (uses GPU-accelerated transforms)
