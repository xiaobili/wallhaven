### Task 2: Create eslint.config.js with no-restricted-imports rule

<read_first>
- `/Volumes/DATA/Code/Vscode/wallhaven/package.json` (for context on project structure)
</read_first>

<action>
Create a new file `eslint.config.js` in the project root with the following content:

```javascript
import js from '@eslint/js'
import pluginVue from 'eslint-plugin-vue'
import tseslint from 'typescript-eslint'

export default [
  js.configs.recommended,
  ...tseslint.configs.recommended,
  ...pluginVue.configs['flat/recommended'],
  {
    files: ['src/views/**/*.{vue,ts}'],
    rules: {
      'no-restricted-imports': ['error', {
        patterns: [{
          group: ['@/stores/*'],
          message: 'Views must import from @/composables, not @/stores. See View → Composable → Store architecture.'
        }]
      }]
    }
  },
  {
    ignores: ['out/**', 'dist/**', 'node_modules/**', 'resources/**']
  }
]
```
</action>

<acceptance_criteria>
- File `eslint.config.js` exists in project root
- File contains `import js from '@eslint/js'`
- File contains `import pluginVue from 'eslint-plugin-vue'`
- File contains `import tseslint from 'typescript-eslint'`
- File contains `'no-restricted-imports': ['error'` pattern
- File contains `group: ['@/stores/*']`
- File contains message about View → Composable → Store architecture
- File contains `ignores: ['out/**', 'dist/**', 'node_modules/**', 'resources/**']`
</acceptance_criteria>
