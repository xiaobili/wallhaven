   ```typescript
   const filteredFavorites = computed(() => {
     if (!selectedCollectionId.value) {
       // "All favorites" mode - deduplicate by wallpaperId, sort newest first
       const seen = new Set<string>()
       return favorites.value
         .slice()
         .sort((a, b) => new Date(b.addedAt).getTime() - new Date(a.addedAt).getTime())
         .filter(f => {
           if (seen.has(f.wallpaperId)) return false
           seen.add(f.wallpaperId)
           return true
         })
     }
     return favorites.value
       .filter(f => f.collectionId === selectedCollectionId.value)
       .sort((a, b) => new Date(b.addedAt).getTime() - new Date(a.addedAt).getTime())
   })
   ```

**Part A: FavoritesPage.vue -- add defineOptions for KeepAlive compatibility**

Per RESEARCH Pitfall 2, Vue 3 `<script setup>` components don't automatically infer a name from the filename. KeepAlive `include` uses the component's explicit `name` option. Without it, KeepAlive silently fails to cache.

Add this inside the `<script setup>` block of FavoritesPage.vue (after the imports):
```typescript
defineOptions({ name: 'FavoritesPage' })
```

This is REQUIRED for KeepAlive to work. The existing cached components (OnlineWallpaper, LocalWallpaper, DownloadWallpaper) presumably already have their names set -- verify by checking if they also use `defineOptions` or a separate `<script>` block. If they use a different pattern, match it. But `defineOptions` is the Vue 3.3+ standard approach with `<script setup>`.
