         - `getActiveCount` is called by `processQueue()` to check `activeDownloads.size`
         - `onDequeue` is called when a task is dequeued and should be executed — this avoids circular dependency (the queue doesn't import from download.handler.ts)

       - **`get length(): number`** — `return this._queue.length`

       - **`has(taskId: string): boolean`** — returns `this._queue.some(item => item.taskId === taskId)`. Used for dedup check (per Pitfall 4 from RESEARCH.md)

       - **`enqueue(item: QueuedDownload): void`** (D-06, D-08):
         - Early return with warning log if `this.has(item.taskId)` (dedup)
         - Push to `_queue`
         - Emit 'waiting' progress event to all windows via `BrowserWindow.getAllWindows()` and `webContents.send('download-progress', ...)` with `state: 'waiting'`, `progress: 0`
         - Call `this.processQueue()`

       - **`processQueue(): void`** (D-06, D-05, D-07):
         - Read `maxConcurrentDownloads` fresh from store: `store.get('appSettings')?.maxConcurrentDownloads ?? 3` (D-15)
         - `while (this._getActiveCount() < maxConcurrent && this._queue.length > 0)`:
           - `shift()` the next item
           - Log dequeue action
           - Call `this._onDequeue(next).catch(err => logHandler(...))` — fire-and-forget
         - After the while loop, iterate remaining queue items and emit progress events updating them with their queue position

       - **`remove(taskId: string): boolean`** (D-09, D-10):
         - Find index via `findIndex(item => item.taskId === taskId)`
         - If found (idx !== -1): `splice(idx, 1)`, log removal, return `true`
         - Otherwise: return `false`
         - NOTE: Do NOT emit progress events here — the caller (PAUSE/CANCEL handler) handles state notification

       - **`clear(): void`** — Reset `_queue = []`, log action. Used for future app shutdown.
