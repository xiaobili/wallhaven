# Plan 24-02: Add Vue Transition to ImagePreview Component

<plan id="24-02">
<objective>Wrap the main image element in a Vue Transition component with dynamic name binding and key to enable slide animations during navigation</objective>

<read_first>
- src/components/ImagePreview.vue (current template structure)
- .planning/phases/24-imagepreview-switch-animation/24-CONTEXT.md (D-01, D-02, D-03 decisions)
- .planning/phases/24-imagepreview-switch-animation/24-RESEARCH.md (section 3.1.B, 3.1.C, 3.1.D)
</read_first>

<truths>
- MUST NOT change existing user operation logic (navigation methods emit same events)
- MUST NOT change existing UI layout (img element remains in same position)
- MUST use Vue 3 built-in Transition component (not CSS class binding)
- Transition name MUST be dynamic based on navigation direction
- :key MUST be imgInfo.id to trigger transition on image change
- NO mode attribute on Transition (simultaneous enter/leave for overlap effect)
- slide-left for prev navigation (new image enters from left)
- slide-right for next navigation (new image enters from right)
- Existing blowUpModal animations must remain unchanged
- img-view container must have position: relative for absolute positioning of leaving element
</truths>

<must_haves>
- Transition component wrapping the main img element
- :key="imgInfo.id" binding on the img element
- :name="transitionName" dynamic binding on Transition
- transitionName ref initialized to 'slide-right'
- navigatePrev() sets transitionName to 'slide-left' before emit
- navigateNext() sets transitionName to 'slide-right' before emit
- position: relative on .img-view container
</must_haves>
