  <action>
    在现有 `src/composables/wallpaper/useWallpaperSetter.ts` 中做以下四个修改：

    **修改 1：添加 `DownloadResult` 接口导出**

    在文件顶部、`UseWallpaperSetterReturn` 接口之前，添加：
    ```typescript
    export interface DownloadResult {
      success: boolean
      filePath: string | null
      error: string | null
    }
    ```

    **修改 2：更新 `UseWallpaperSetterReturn` 接口**

    在现有 `loading` 和 `setWallpaper` 基础上，添加两个新成员：
    ```typescript
    export interface UseWallpaperSetterReturn {
      loading: Ref<boolean>
      setWallpaper: (imagePath: string) => Promise<boolean>
      downloadWallpaperFile: (imgItem: WallpaperItem) => Promise<DownloadResult>
      setBgFromUrl: (imgItem: WallpaperItem) => Promise<void>
    }
    ```

    需要在文件顶部添加 `import type { WallpaperItem } from '@/types'`。保持现有的 Vue import 不变。

    **修改 3：在 `useWallpaperSetter` 函数体内添加 `useSettings` 调用**

    在函数体顶部，在 `const loading = ref(false)` 之后添加：
    ```typescript
    const { settings, selectFolder, update: updateSettings } = useSettings()
    ```

    保持现有的 `const { showError, showSuccess } = useAlert()` 不变。

    **修改 4：添加 `downloadWallpaperFile` 和 `setBgFromUrl` 方法**

    在 `setWallpaper` 方法之后（`try/catch/finally` 块的 `finally` 之后）、`return` 语句之前，添加两个新方法：

    `downloadWallpaperFile` — 这是一个 async 方法，接收 `imgItem: WallpaperItem`，返回 `Promise<DownloadResult>`。逻辑（完全来自两个 View 的现有实现，字节级相同）：
    1. 读取 `settings.value.downloadPath`
    2. 如果没有设置下载目录，调用 `selectFolder()` — 如果用户取消选择，返回 `{ success: false, filePath: null, error: '未选择下载目录' }`
    3. 选择后，调用 `updateSettings({ downloadPath: selectResult.data })` 保存设置
    4. 从 `imgItem.path` 提取扩展名（`/\.(jpg|jpeg|png|gif|bmp|webp)$/i`），默认 `.jpg`
    5. 生成文件名 `wallhaven-${imgItem.id}${ext}`
    6. 动态导入 `await import('@/clients')`，取出 `electronClient`
    7. 调用 `electronClient.downloadWallpaper({ url: imgItem.path, filename, saveDir: downloadPath })`
    8. 返回 `{ success: result.success, filePath: result.data || null, error: result.error?.message || null }`

    `setBgFromUrl` — 这是一个 async 方法，接收 `imgItem: WallpaperItem`，返回 `Promise<void>`。逻辑：
    1. 调用 `const downloadResult = await downloadWallpaperFile(imgItem)`
    2. 如果 `!downloadResult.success || !downloadResult.filePath`，调用 `showError('下载壁纸失败: ' + (downloadResult.error || '未知错误'))` 并 `return`
    3. 调用 `await setWallpaper(downloadResult.filePath)`
    4. 错误处理：`try/catch` 包裹以上三步，catch 中 `console.error('设置壁纸错误:', error)` 和 `showError('设置壁纸失败: ' + error.message)`

    **不修改** 现有的 `setWallpaper` 方法 — `loading` ref 继续只跟踪 `setWallpaper` 的执行，不覆盖 download 阶段（per Pitfall 2 分析）。

    **不需要** 修改现有的 `import { ref, type Ref } from 'vue'` — 保持原样。新增一个 `import type { WallpaperItem } from '@/types'`。

    **必须保持** 动态 import `await import('@/clients')` 的方式，与两个 View 中一致（不是静态 import），避免在 vitest 环境中报错（per Pitfall 3 分析）。
