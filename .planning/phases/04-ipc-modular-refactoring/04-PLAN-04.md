## Goal

Delete the old handlers.ts file and update main/index.ts to use the new modular handlers. This is the critical integration step that completes the IPC refactoring.

## Task 1: Update main/index.ts to Use New Handlers

<read_first>
- electron/main/index.ts (current state)
- electron/main/ipc/handlers/index.ts (new registration entry point)
</read_first>

<acceptance_criteria>
- File `electron/main/index.ts` imports `verifyHandlers` from `'./ipc/handlers/index'`
- File `electron/main/index.ts` NO LONGER has `import './ipc/handlers'` line
- File `electron/main/index.ts` calls `verifyHandlers()` inside `app.whenReady().then()` callback
- `verifyHandlers()` is called after `createWindow()`
- TypeScript compilation succeeds
</acceptance_criteria>

<action>
Modify `electron/main/index.ts`:

1. **Replace the handler import (line 30)**:
   
   **BEFORE:**
   ```typescript
   // 导入IPC handlers
   import './ipc/handlers'
   ```

   **AFTER:**
   ```typescript
   // 导入IPC handlers (modular)
   import { verifyHandlers } from './ipc/handlers/index'
   ```

2. **Add verification in app.whenReady callback (around line 150)**:

   **Find the app.whenReady().then() block and add verifyHandlers() call:**
   ```typescript
   app.whenReady().then(() => {
     // 注册自定义协议（必须在 app.whenReady 之后）
     registerLocalFileProtocol()

     // Set app user model id for windows
     electronApp.setAppUserModelId('com.wallhaven')

     // Default open or close DevTools by F12 in development
     // and ignore CommandOrControl + R in production.
     app.on('browser-window-created', (_: any, window: any) => {
       optimizer.watchWindowShortcuts(window)
     })

     createWindow()

     // Verify all IPC handlers are registered
     verifyHandlers()

     app.on('activate', function () {
       // On macOS it's common to re-create a window in the
       // dock icon is clicked and there are no other windows open.
       if (BrowserWindow.getAllWindows().length === 0) createWindow()
     })
   })
   ```

**IMPORTANT:** The `verifyHandlers()` call MUST be after `createWindow()` to ensure the app is fully initialized before verification runs.
</action>
