# PLAN 05: Add Error Handling for Resume Failures

## Objective

Enhance error handling in the `resumeDownload` method to handle specific error cases from the main process and update the task state accordingly.

## Context

- Phase 7 defined error codes in main process: `RESUME_FILE_NOT_FOUND`, `RESUME_STATE_CORRUPTED`, `RESUME_SERVER_UNSUPPORTED`, `RESUME_INVALID_OFFSET`
- When resume fails, the task should remain in `paused` state and show appropriate error message
- Some errors may require removing the task from the list (e.g., file not found)

<name>Enhance resumeDownload error handling with specific error codes</name>
<read_first>
- src/composables/download/useDownload.ts (resumeDownload method from PLAN 02)
- src/shared/types/ipc.ts (error codes pattern)

<action>
Update the error handling section in the `resumeDownload` method to handle specific error codes.

The current error handling from PLAN 02 is:
```typescript
if (!result.success) {
  task.state = 'paused'
  showError(result.error?.message || '恢复下载失败')
  return false
}
```

Replace with:
```typescript
if (!result.success) {
  const errorCode = result.error?.code

  // 根据错误类型处理
  if (errorCode === 'RESUME_FILE_NOT_FOUND' || errorCode === 'RESUME_STATE_CORRUPTED') {
    // 临时文件丢失或状态损坏，从列表移除
    showError(`下载任务已失效: ${result.error?.message || '文件不存在'}`)
    const index = store.downloadingList.findIndex((item) => item.id === id)
    if (index !== -1) {
      store.downloadingList.splice(index, 1)
    }
    return false
  }

  // 其他错误，保持暂停状态允许重试
  task.state = 'paused'
  showError(result.error?.message || '恢复下载失败')
  return false
}
```
</action>

- When these errors occur, the task is removed from `store.downloadingList`

## Notes

- Error codes are defined in the main process and passed through IPC
- The `RESUME_SERVER_UNSUPPORTED` error (server returns 200 instead of 206) is handled by the main process - it triggers a fresh download, so we don't need special handling in renderer
- Removing tasks for `RESUME_FILE_NOT_FOUND` and `RESUME_STATE_CORRUPTED` prevents showing permanently broken tasks
