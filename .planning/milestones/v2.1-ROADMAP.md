## Overview

为下载功能添加完整的断点续传能力，用户中断后可从暂停点继续下载，应用重启后自动恢复未完成的下载任务。

**Core Value:** 断点续传，下载无忧 — 大文件下载不再担心中断，随时随地暂停恢复

### Phase 7: Main Process Implementation

**Focus:** Range requests, state persistence, temp file handling

**Requirements:** INFR-02, INFR-03, CORE-02

**Plans:**
1. ✅ Add state persistence utility functions
2. ✅ Modify pause handler and add progress state persistence
3. ✅ Implement RESUME_DOWNLOAD_TASK handler with Range support
4. ✅ Implement GET_PENDING_DOWNLOADS handler

**Key Implementation:**
- Atomic state file writes (write to .tmp, then rename)
- Throttled state persistence (every 5s or 10MB)
- HTTP Range request with `bytes={offset}-` header
- Response status handling: 206 (append) vs 200 (restart)

## Key Decisions

| Decision | Outcome |
|----------|---------|
| Direct Range request (no HEAD pre-detection) | ✓ Works well, add notification on 200 response |
| Atomic state file writes | ✓ Prevents corruption |
| Throttled persistence (5s/10MB) | ✓ Balances performance and reliability |
