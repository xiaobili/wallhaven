## v4.0 Requirements

### 下载队列与并发控制

- [x] **DL-01**: 应用下载时遵循设置页的"并行下载数"配置，不超过 N 个下载任务同时进行
- [x] **DL-02**: 超出并发限制的下载任务自动进入等待队列，当前下载完成后自动开始下一个
- [x] **DL-03**: 用户修改"并行下载数"设置后，新值立即生效，无需重启应用
- [x] **DL-04**: 并发数从高调低时，超出的活跃下载不会被中断（等待新任务时按新限制执行）

### 错误分类与重试退避

- [x] **DL-05**: 下载失败时自动判断错误类型，临时错误（网络超时、连接重置、5xx、429）自动重试
- [x] **DL-06**: 永久错误（404、403、401 等客户端错误）不重试，直接标记为失败
- [x] **DL-07**: 重试采用指数退避策略，每次重试等待时间递增（含 jitter 防抖动）
- [x] **DL-08**: 重试次数上限为 3 次，超过后标记为失败
- [x] **DL-09**: 重试中的下载依然占用并发槽位，不额外增加连接数

### 重试状态展示

- [x] **UI-01**: 下载列表中显示正在重试的下载任务，标明"重试中 (第X次/共3次)"
- [x] **UI-02**: 下载列表显示下次重试前的倒计时
- [x] **UI-03**: 最终失败的下载显示"下载失败 — 已重试 X 次"

## Deferred Items

- **DL-NEXT**: 最大重试次数用户可配置（当前为编译时常数 3）
- **DL-NEXT**: 将下载队列持久化到磁盘，应用重启后恢复等待队列
- **DL-NEXT**: 应用重启后重置重试计数（临时性问题通常已解决）
