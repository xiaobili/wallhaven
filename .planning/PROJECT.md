# Wallhaven 壁纸浏览器 — 架构重构项目

## 项目概述

这是一个针对现有 Wallhaven 壁纸浏览器应用的**纯重构项目**，目标是优化代码架构、增强可读性和鲁棒性，同时保持所有现有功能和用户体验不变。

## 核心价值

**外观行为零感知，内部架构大升级**

在不改变任何用户操作逻辑、界面布局、UI 显示的前提下，完成代码架构的全面优化，为未来功能扩展奠定坚实基础。

## 约束条件

### 不可变更（硬约束）

- **用户操作逻辑**：所有交互行为保持原样
- **界面布局**：组件结构和 DOM 结构不变
- **UI 显示效果**：样式、动画、视觉效果不变
- **功能行为**：所有现有功能的输入输出行为不变
- **API 兼容**：IPC 通道名称和消息格式保持向后兼容

### 可变更（重构范围）

- **代码内部架构**：模块划分、层次结构
- **类型定义**：消除 `any`，添加完整类型
- **错误处理机制**：添加错误边界、全局错误处理
- **代码组织**：文件拆分、命名规范、提取公共逻辑

## 需求范围

### 已验证（现有功能）

以下功能已在代码库中实现并验证：

- ✓ 壁纸搜索 — 关键词、分类、纯度、分辨率等筛选
- ✓ 图片预览 — 大图预览、全屏查看、动画效果
- ✓ 无限滚动 — 自动加载更多、节流优化
- ✓ 下载管理 — 批量下载、进度追踪、暂停/恢复
- ✓ 本地壁纸管理 — 文件夹浏览、缩略图生成
- ✓ 应用设置 — 下载路径、API Key、壁纸适配模式
- ✓ 桌面壁纸设置 — 多种适配模式

### 活跃（重构目标）

**架构优化（优先级 1）**：

- [ ] IPC 模块化拆分 — 将 866 行 handlers.ts 拆分为领域模块
- [ ] Composables 抽象 — 提取 Alert 等重复逻辑为可复用组合式函数
- [ ] 服务层抽象 — 为 Electron API 创建统一服务层
- [ ] Store 重构 — 分离数据访问逻辑，实现 Repository 模式

**鲁棒性增强（优先级 2）**：

- [ ] 类型安全强化 — 消除 60+ 处 `any` 类型，添加完整接口定义
- [ ] 错误边界与处理 — 添加 Vue Error Boundary、全局错误处理

**可读性提升（优先级 3）**：

- [ ] 代码规范化 — 统一命名规范、清理死代码、提取工具函数
- [ ] 注释与文档 — 添加 JSDoc 注释、更新技术文档
- [ ] 文件组织优化 — 统一代码风格、优化文件结构

### 超出范围

| 功能 | 原因 |
|------|------|
| 新功能开发 | 本次为纯重构项目 |
| UI/UX 变更 | 保持用户体验一致 |
| 性能优化 | 非本次重构重点，可在后续迭代 |
| 安全加固 | API Key 加密等可在后续迭代 |
| 测试覆盖 | 可在架构稳定后添加 |

## 技术背景

### 当前技术栈

- **桌面框架**：Electron v41.2.2
- **前端框架**：Vue 3.5.32 (Composition API)
- **构建工具**：electron-vite 5.0.0 / Vite 7.3.2
- **语言**：TypeScript 6.0.0
- **状态管理**：Pinia 3.0.4
- **路由**：Vue Router 5.0.4
- **HTTP 客户端**：Axios 1.15.0

### 已识别的技术债务

**高优先级**：
- IPC 处理器单文件 866 行，缺乏模块化
- Alert 状态管理在 4+ 个组件中重复
- 60+ 处 `any` 类型使用
- 无 Vue Error Boundary

**中优先级**：
- Electron API 调用散落在各组件中
- Store 直接调用 IPC，缺乏抽象层
- 死代码（测试/演示组件未集成）

**低优先级**：
- 缺少 JSDoc 注释
- 文件组织可进一步优化
